VisionGate AI - FAQs
---

### 1. Difficulties faced
- **Dual-View Image Processing & State Management:** Synchronising two concurrent large image streams (representing Left and Right container views) and passing them simultaneously to a language/vision model without heavy timeouts. Ensuring that uploading two images did not erroneously create two separate SQLite logs, but merged into a single unified Gate Incident record.
- **Multi-modal Documentation Cross-referencing:** Designing a prompt infrastructure that could safely ingest an unstructured PDF manifest via PyPDF2, append it alongside raw image pixels, and force the AI to compare the visual state against the declared constraints without breaking the strict JSON schema required for our bounding boxes.
- **Prompt Engineering Robust JSON Extraction:** Ensuring the Vision API consistently returned valid, parseable JSON arrays including normalized bounding box matrices in every single inference.

### 2. What can be extended in future
- **Strict Edge-ML Architecture:** Fully detaching from cloud-based AI calls by actively running INT8 quantized YOLOv8 (for damage bounding) and PaddleOCR (for ISO tracking) directly on NVIDIA Jetson or edge-IoT racks installed locally at the gate barriers. 
- **Hardware Integration:** Hooking up our output `recommended_actions` via GPIO arrays to physically lock and open gate barriers based on the system's output status.
- **Live Video Streaming Analysis:** Evolving from uploading snapshots to ingesting an RTSP CCTV video stream and grabbing container frames autonomously using motion detection triggers.

### 3. How your project is better than existing solutions in the market
- **Drastic Processing Speed:** A traditional container triage requires manual clipboard documentation that takes 3-5 minutes per truck. Our pipeline is mathematically modelled to clear a truck in ~14 seconds.
- **Contextual Cognitive Checking:** Current market solutions are simple computer-vision boxes that just say "Rust detected". VisionGate combines multiple modalities: it contextualizes the "Rust" against live Port Weather (predicting water-ingress risk in a storm) and checks visual Hazmat placards against PDF documentation manifests dynamically.
- **Integrated AI Yard Copilot:** We don't just provide a dashboard; we provide an embedded conversational AI grounded directly into the local SQLite database context so operators can query logistics natively.

### 4. Database schema
We employed an embedded SQLite architecture (`visiongate.db`) utilizing the python `sqlite3` library for fast, serverless storage suited for edge deployments. The schema consists of two primary tables:
- **`container_logs` Table:**
  - `id` (INTEGER PRIMARY KEY)
  - `timestamp` (TEXT)
  - `iso_code` (TEXT) 
  - `damage_status` (TEXT) - e.g. CLEAR, MINOR_DAMAGE, WARNING, CRITICAL
  - `severity` (TEXT)
  - `recommended_action` (TEXT)
  - `location` (TEXT) - e.g. terminal geography
  - `inspection_type` (TEXT) - 'structural' or 'thermal' logic separation
- **`ship_delays` Table:** 
  - Tracks specific delay metrics with fields for `ship_name`, `terminal`, `delay_minutes`, `driver_contact` and `sms_sent`.

### 5. Motivation of your project
The project targets DP World's "Our World, Our Future" ESG framework. Port gate queues are massive bottlenecks that create long lines of commercial trucks. Every minute a truck idles waiting for a manual clerk, it burns diesel and accelerates Scope 3 carbon emissions locally. By deploying AI to slash inspection time by 95%, we drastically curb idling CO2 emissions while improving the safety net for dangerous, illicitly loaded hazmat cargo.

### 6. For each technology you used, why you used that and not competitor technology
- **Streamlit (over React/Node):** We wanted ultra-rapid iteration in a pure Python environment because we are heavy on data manipulation and ML endpoints. Streamlit handles Numpy/Pandas matrices inherently and makes drawing dynamic Pillow bounding boxes significantly easier than building separate REST JSON bridges to a frontend framework.
- **Gemini 2.5 Flash API (over OpenAI GPT-4o):** Gemini currently offers superior cost-to-speed ratios for heavy Multi-modal RAG operations. Sending multiple high-resolution images combined with large extracted PDF context blocks required a massive context window and rapid processing which Gemini excelled in.
- **SQLite (over PostgreSQL/MySQL):** A heavy DB infrastructure was unnecessary overkill for a decentralized "edge-terminal" simulation. SQLite retains strict relational integrity and ACID compliance with zero configuration. 
- **PyPDF2 (over separate Document/Vision models):** We extracted native string characters directly from digital PDFs natively instead of using OCR on document images. This accelerates inference and completely zeroes out LLM text hallucinations.

### 7. What complex scenario you faced while making the project
The most complex architectural hurdle was handling the "Thermal Dashboard" vs "Gate Inspector" data overlap. We initially faced issues where thermal metric calculations (temperature and heat signatures) were polluting the structural AI dashboard calculations. Solving this required writing a custom database migration directly in our `db_utils.py` script to introduce the `inspection_type` field and dynamically sandboxing aggregate SUMs and COUNTs across millions of rows using conditional SQLite logic so dashboards accurately represented their domain without collision.

### 8. What did you fail to add in the project
- **Blockchain Digital Footprints:** We simulated "immutable audit logs" inside the PDF generation tool, but we did not implement an actual Hyperledger Fabric network to store the cryptographic hashes of our container records to prevent log manipulation.
- **Live TOS API Link:** Because we could not access a true bi-directional port API like CARGOES TOS during a hackathon prototype cycle, all external events (smart routing) are internally sequestered logs.
- **Live CCTV Feeds:** As mentioned before, native browser webcam integration (via WebRTC) for 24fps CV scanning was excluded to focus purely on perfecting the single-shot multimodal intelligence layer.
